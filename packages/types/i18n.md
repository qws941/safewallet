# Korean (ko-KR) Localization

## Overview

The `i18n` module in `@safetywallet/types` provides a complete, type-safe Korean localization system for the SafeWork2 Worker App.

**Features:**
- 300+ localized strings organized by feature section
- Full TypeScript support with `as const` for type safety
- Single source of truth for all UI text
- No runtime dependencies

## File Structure

```
packages/types/src/i18n/
├── index.ts          # Barrel exports
└── ko.ts             # Korean localization object (300+ strings)
```

## Usage

### Importing the localization object

```typescript
import { ko } from '@safetywallet/types';

// Access any string with full type safety
const title = ko['login.title']; // '안전지갑'
const errorMsg = ko['login.errorUserNotFound']; // '사용자를 찾을 수 없습니다'
```

### In React components

```typescript
'use client';

import { ko } from '@safetywallet/types';

export function LoginPage() {
  return (
    <div>
      <h1>{ko['login.title']}</h1>
      <p>{ko['login.description']}</p>
      <input placeholder={ko['login.phonePlaceholder']} />
      <button>{ko['login.submitButton']}</button>
    </div>
  );
}
```

### With dynamic content (string interpolation)

```typescript
// For strings with variables, use template literals
const fileCount = 5;
const message = `${fileCount}개의 파일이 업로드되었습니다`;

// Or create a helper function:
function getFileUploadMessage(count: number): string {
  return `${count}개의 파일이 업로드되었습니다`;
}
```

## Type Safety

The `ko` object is exported with `as const`, providing:

1. **Literal types**: Each key is a literal string type
2. **IntelliSense**: Full autocomplete in IDEs
3. **Type checking**: TypeScript catches missing or typo'd keys at compile time

```typescript
// ✅ Type-safe - autocomplete works
const x = ko['login.title'];

// ❌ Type error - key doesn't exist
const y = ko['login.foo']; // Error: Property 'login.foo' does not exist
```

## Adding New Strings

When adding new UI text:

1. Identify the feature section (e.g., `login`, `posts`, `actions`)
2. Choose a descriptive key name in camelCase (e.g., `errorUserNotFound`)
3. Add to `ko.ts` in the appropriate section:

```typescript
'login.myNewKey': '새로운 한글 텍스트',
```

4. Run `npm run typecheck` to verify
5. Use in component: `ko['login.myNewKey']`

## Maintenance

### Verifying all strings are covered

```bash
# Type check the types package
cd packages/types
npm run typecheck

# Should see: No errors
```

### Extracting new strings from components

If you find hardcoded Korean text in a component:

1. Identify the string and its context
2. Add an appropriate key to `ko.ts`
3. Replace the hardcoded text with `ko['section.keyName']`
4. Run typecheck

### Migration Guide (for existing code)

Before:
```typescript
export function LoginPage() {
  return <h1>안전지갑</h1>;
}
```

After:
```typescript
import { ko } from '@safetywallet/types';

export function LoginPage() {
  return <h1>{ko['login.title']}</h1>;
}
```

## Advanced Usage

### Creating a locale-aware hook (future)

```typescript
// hooks/useI18n.ts
import { ko } from '@safetywallet/types';

export function useI18n() {
  // Currently ko is the only locale
  return ko;
}

// Usage in component
export function MyComponent() {
  const i18n = useI18n();
  return <h1>{i18n['login.title']}</h1>;
}
```

### Supporting multiple locales (future)

The structure is designed to easily add more locales:

```typescript
// packages/types/src/i18n/index.ts
export { ko, type Ko } from './ko';
export { en, type En } from './en';
export { ja, type Ja } from './ja';

import { ko } from './ko';
import { en } from './en';
import { ja } from './ja';

export const i18n = { ko, en, ja };
export type I18n = typeof i18n;
```

## Export Details

### From `@safetywallet/types`:

```typescript
// Direct import
import { ko } from '@safetywallet/types';

// Or typed import
import type { Ko } from '@safetywallet/types';

// Or the i18n namespace
import { i18n } from '@safetywallet/types';
```

### Type definitions

```typescript
// Ko type represents the shape of the localization object
type Ko = typeof ko;

// I18n type represents all available locales (currently just ko)
type I18n = typeof i18n;
```
